Alguns dos principais documentos brasileiros utilizam as mesmas bases de cálculo para seus dígitos verificadores --representados nos exemplos ao lado pelo sinal de interrogação--, que são os algorítmos conhecidos como Modulus 10 e Modulus 11. Como em muitas implementações para cálculo desses dígitos não separa estes algorítmos do cálculo em si, resultando na repetição de código, este projeto em Javascript, surge para mostrar como essa abordagem é mais fácil e até mesmo mais lógica.

Com esta página de exemplo objetiva-se mostrar o funcionamento dessa solução, bem como chamar a atenção de programadores para a validação de dados do usuário. A maioria dos sistemas obriga os usuários a digitar documentos --até mesmo números de telefone-- da forma com que esperam: somente os números ou com a formatação adequada. Acontece que já existe tecnologia para permitir que os dados sejam digitados da forma que o usuário quiser e o programa processar essa entrada adequadamente.

Os campos ao lado podem ser preenchidos como quiser: somente números, formatados com pontos e traços, com espaços ou quaisquer caracteres não numéricos. Somente os números serão levados em consideração. Caso todos os números esperados sejam digitados --e.g., 11 para o CPF--, o sistema verificará se eles formam um número válido de documento. Se positivo, ele será impresso no label próximo, formatado e na cor verde. Caso contrário ficará na cor vermelha. Se mais números forem digitados, apenas os X primeiros serão considerados, onde X é o limite para cada documento. Se menos números --ou nenhum-- forem digitados, o sistema preencherá os restantes aleatoriamente e calculará os dígitos verificadores, apresentando o número em preto no label.
